# Тестовое задание: Автотесты на Selenium + Java (SauceDemo)

Проект содержит 10 автоматизированных UI-тестов для сайта [SauceDemo](https://saucedemo.com).

## 🛠 Стек технологий
* **Язык:** Java 17 / JDK
* **Фреймворк тестирования:** JUnit 5
* **Инструмент автоматизации:** Selenium WebDriver (с использованием встроенного Selenium Manager)
* **Сборщик проекта:** Maven

## 📊 Список автотестов (10 тестов)
1. `smokeTest1TitleCheck` (Smoke, Позитивный) — проверка доступности главной страницы по заголовку.
2. `smokeTest2SuccessfulLogin` (Smoke, Позитивный) — проверка успешной авторизации под standard_user.
3. `negativeTestWrongPassword` (Негативный) — проверка валидации при вводе неверного пароля.
4. `negativeTestEmptyUsername` (Негативный) — проверка валидации при пустом поле логина.
5. `bugTestProblemUserSorting` (Находит баг) — тест намеренно падает, фиксируя дефект сортировки у problem_user.
6. `positiveTestAddToCart` (Позитивный) — проверка добавления товара в корзину и обновления счетчика.
7. `positiveTestRemoveFromCart` (Позитивный) — проверка удаления товара и исчезновения счетчика корзины.
8. `positiveTestOpenProductPage` (Позитивный) — проверка перехода в детальную карточку товара.
9. `positiveTestSidebarMenu` (Позитивный) — проверка открытия бокового бургер-меню.
10. `positiveTestFooterCopyright` (Позитивный) — проверка корректности текста копирайта в футере.

  Баг-репорт №3 (Альтернативный)
  
  ID: BG-002
  
  Тема: Не работает кнопка «Remove» (Удалить из корзины) для товара "Sauce Labs Fleece Jacket" под пользователем problem_user
  
  Компонент: Каталог товаров / Корзина
  
  Серьезность (Severity): Major (Значительная) — критический функционал корзины частично заблокирован
  
  
  Приоритет (Priority): High (Высокий)
  
  Окружение: Google Chrome (Версия: 149.0.7827.156), saucedemo.com
  
  Шаги воспроизведения:
  
  Перейти на сайт https://saucedemo.com.
  
  Авторизоваться, используя логин problem_user и пароль secret_sauce.
  
  Найти в каталоге товар "Sauce Labs Fleece Jacket" (Куртка).
  
  Нажать на кнопку "Add to cart" для этого товара (убедиться, что на иконке корзины вверху появилась цифра "1", а кнопка сменила текст на "Remove").
  
  
  Нажать на изменившуюся кнопку "Remove" у этого же товара.
  
  Посмотреть на счетчик корзины и статус кнопки.
  
  Фактический результат:
  
  Кнопка никак не реагирует на нажатие. Текст кнопки остается "Remove", товар не удаляется из корзины, а на красном значке корзины сверху по-прежнему отображается цифра "1". Пользователь заблокирован в возможности отменить выбор товара с главной страницы.
  
  Ожидаемый результат:
  
  Товар успешно удаляется из корзины. Кнопка меняет свой текст обратно на "Add to cart", а красный счетчик на иконке корзины исчезает (потому как в корзине становится 0 товаров).
